## Top 3 Priority Fixes

No priority fixes required. All 12 deviations targeted by this phase have been resolved. Three minor observations are noted below for awareness — none deduct from the score.

1. **UI-SPEC internal inconsistency on tabIndex** — The spec (line 222) states "No tabIndex needed" for card focus rings, which is technically incorrect: `<div>` elements are not keyboard-focusable without `tabIndex={0}`. The implementation (plan 02) correctly contradicts the spec and adds `tabIndex={0}` to all three card wrappers. The spec should be updated to reflect the correct guidance so future phases do not regress this fix. Concrete fix: update line 222 of 05.1-UI-SPEC.md to read "tabIndex={0} required on div cards".

2. **Contact page heading mobile size** — `contact/page.tsx` line 34 uses `text-xl lg:text-4xl`, which maps to Heading (28px mobile) scaling to Display (36px desktop). This is correct per the spec correction, but means the contact page H1 is visually smaller than other page H1s on mobile where all other pages consistently use `text-4xl`. No spec violation — the spec mandated `text-xl` to replace `text-[1.75rem]` — but this creates a minor visual inconsistency across public pages at mobile breakpoints. Worth reviewing when real content is loaded.

3. **Empty-state on mon-histoire lacks font-body class** — `mon-histoire/page.tsx` line 62 renders the empty state as `<p className="text-gris-ardoise text-center">` without `text-base font-body`, diverging from the body copy pattern used on all other empty states. Low visual impact since `text-base` is the default, but the missing token class is inconsistent with the copy contract style.
