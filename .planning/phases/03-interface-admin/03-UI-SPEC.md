# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Interface Admin.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note

Phase 3 produces **no new public-facing pages**. All UI work is inside the Payload CMS admin panel at `/admin`. The public site design system (Tailwind v4 tokens, Montserrat/Inter fonts, "Energie & Performance" palette) is already complete and must not be touched in this phase.

The design contract here governs:
1. The **custom dashboard widget** (`DashboardWidget.tsx`) injected via `admin.components.beforeDashboard`
2. **Admin panel configuration** choices: labels, field descriptions, Lexical toolbar, sidebar grouping

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (manual) | No `components.json` found; Payload admin ships its own design system |
| Preset | not applicable | Phase 3 uses Payload admin theme, not shadcn |
| Component library | Payload admin built-in | All admin UI components come from Payload CMS |
| Icon library | Payload admin built-in | No additional icon library for admin phase |
| Font (public site) | Montserrat (headings) + Inter (body) | `src/app/globals.css` — `@theme` tokens D-06, D-07 |
| Admin theme | Payload default (light) | Payload admin handles its own theming |

**Dashboard widget styling approach:** Use Payload CSS custom properties (`var(--theme-elevation-50)`, `var(--theme-elevation-100)`, `var(--theme-text)`) for the widget to match the admin aesthetic. Fall back to neutral grays (`#f8fafc`, `#1e293b`) if a Payload variable is unavailable at render time. Do **not** import Tailwind classes into the admin widget — Tailwind is scoped to the public site and is not guaranteed to be available in Payload admin context.

## Spacing Scale

The 8-point spacing scale applies to the custom dashboard widget only. All other admin spacing is controlled by Payload's own stylesheet.

| Token | Value | Usage in DashboardWidget |
|-------|-------|--------------------------|
| xs | 4px | Icon gaps, tight inline spacing |
| sm | 8px | Counter label margin-top |
| md | 16px | Card internal padding |
| lg | 24px | Widget section padding (padding: 1.5rem) |
| xl | 32px | Grid gap between counter cards |
| 2xl | 48px | Widget bottom margin (marginBottom: 2rem) |

Exceptions:
- Touch targets on quick-action links: minimum 44px height to comply with WCAG 2.1 AA (applies inside the admin panel which Antoine uses on desktop, but enforced as good practice)

---

## Typography

Typography for the **DashboardWidget** only. Admin panel body typography is Payload's own.

Weights used: 600 (semibold) + 700 (bold) — exactly 2 weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Counter number | 32px (2rem) | 700 (bold) | 1.2 | Programme/resultat count — the most prominent element |
| Counter label | 14px (0.875rem) | 600 (semibold) | 1.4 | "Programmes", "Resultats avant/apres", "Pages a editer" |
| Quick-action link | 14px (0.875rem) | 600 (semibold) | 1.4 | "+ Ajouter", "Modifier" — clickable labels inside counter cards |
| Widget heading | 20px (1.25rem) | 600 (semibold) | 1.3 | "Bienvenue, Antoine" — optional greeting above counters |

Font family for DashboardWidget: inherit from Payload admin (`font-family: inherit`) — do not override.

## Interaction Contract

### Lexical editor toolbar (D-07)

Restricted to exactly these 5 features — nothing else:

| Feature | Payload class | Rationale |
|---------|---------------|-----------|
| Bold | `BoldFeature()` | Basic emphasis |
| Italic | `ItalicFeature()` | Secondary emphasis |
| Unordered list | `UnorderedListFeature()` | Bullet lists for objectives |
| Ordered list | `OrderedListFeature()` | Numbered sequences |
| Link | `LinkFeature()` | External references |
| Fixed toolbar | `FixedToolbarFeature()` | Always-visible bar — no floating popover confusion |

Explicitly excluded: headings, horizontal rule, tables, code blocks, color pickers, block elements, upload/image embeds inside richText. These are removed to prevent Antoine from breaking the public site layout.

### Sidebar navigation (D-10, D-11)

| Collection | Visible in sidebar | Group | Create button |
|------------|-------------------|-------|---------------|
| Programmes | Yes | "Contenu" | Shown |
| Resultats | Yes | "Contenu" | Shown |
| PageContent (Contenu des pages) | Yes | "Contenu" | Hidden — `access.create: () => false` |
| Media | Yes | "Contenu" | Shown |
| Users | No — `admin.hidden: true` | — | Hidden |

Password change path for Antoine: profile menu icon (top right of Payload admin, built-in) > "Mon compte" > change password.

### PageContent list view (D-06)

| Property | Value |
|----------|-------|
| Default sort | `page` (groups sections by page visually) |
| Default columns | `page`, `section`, `titre` |
| Searchable fields | `page`, `section`, `titre` |
| Filter UX | Payload built-in filter panel on `page` select field — no extra config needed |

### Upload constraints (D-08, D-09)

| Property | Value |
|----------|-------|
| Server-side limit | 5 MB (`upload.limits.fileSize: 5_000_000`) |
| Vercel caveat | `clientUploads: true` bypasses server validation (Payload bug #12671). Limit is informational for Vercel deployments — enforced via field descriptions. |
| Accepted formats | JPEG, PNG, WebP (existing Media collection config) |
| WebP conversion | Via `next/image` at render time (D-08) — no server-side conversion in admin |
