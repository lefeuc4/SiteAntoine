<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Vercel project setup + domain configuration</name>
  <files></files>
  <action>Human action required — Vercel project creation, environment variable configuration, and DNS setup at OVH. See how-to-verify for detailed step-by-step instructions.</action>
  <what-built>Complete site with SEO, RGPD compliance, redirects — ready for production deployment</what-built>
  <how-to-verify>
    **Step 1: Create Vercel Project (if not exists)**
    1. Go to https://vercel.com/new
    2. Import the GitHub repository for SiteAntoine
    3. Framework preset: Next.js (auto-detected)
    4. Add these environment variables in the Vercel project settings:
       - `DATABASE_URI` = (Neon pooled connection string from .env.local — hostname must contain `-pooler`)
       - `PAYLOAD_SECRET` = (value from .env.local)
       - `BLOB_READ_WRITE_TOKEN` = (value from .env.local)
       - `NEXT_PUBLIC_SERVER_URL` = `https://antoineprofit.com`
       - `RESEND_API_KEY` = (value from .env.local)
       - `CONTACT_EMAIL` = (value from .env.local)
    5. Click Deploy — wait for build to succeed
    6. Verify the site works on the `.vercel.app` preview URL

    **Step 2: Custom Domain (per D-14)**
    1. In Vercel Dashboard > Project > Settings > Domains, add `antoineprofit.com` and `www.antoineprofit.com`
    2. Vercel will show the DNS records to configure
    3. In OVH DNS Manager for antoineprofit.com:
       - Set A record for `@` to the IP Vercel provides
       - Set CNAME for `www` to `cname.vercel-dns.com`
       - (Optional) Lower TTL to 300s before cutover for faster propagation
    4. Wait for DNS propagation (usually 1-2 hours, up to 48h)
    5. Vercel automatically provisions SSL certificate once DNS resolves

    **Step 3: Verify Production**
    1. Visit https://antoineprofit.com — site loads with HTTPS
    2. Visit https://antoineprofit.com/mentions-legales — page renders
    3. Visit https://antoineprofit.com/index.php/about-me — redirects to /mon-histoire
    4. Check cookie banner appears on first visit (incognito window)
    5. View page source — meta description and OpenGraph tags present
  </how-to-verify>
