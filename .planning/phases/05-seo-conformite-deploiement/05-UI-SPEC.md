## New UI Components (Phase 5 only)

### CookieBanner

A persistent bottom bar, client component using localStorage to track consent state.

**Layout:**
- Fixed to bottom of viewport: `position: fixed; bottom: 0; left: 0; right: 0; z-index: 50`
- Full-width bar, NOT a modal or popup
- Single row on desktop: text left, buttons right
- Two rows on mobile: text, then buttons below

**Anatomy:**
```
[  Nous utilisons des outils d'analyse (Vercel Analytics)...  [Refuser] [Accepter]  ]
```

**Behavior:**
- Renders only when localStorage key `cookie-consent` is absent
- On "Accepter": set `cookie-consent=accepted` in localStorage, hide banner
- On "Refuser": set `cookie-consent=refused` in localStorage, hide banner
- On revisit: read localStorage, do not render if key exists
- No animation required — simple conditional render

**Padding:** `px-6 py-3` (24px horizontal, 12px vertical)
**Button spacing:** `gap-3` (12px) between the two buttons
**Banner max width:** unconstrained (full viewport width)
**Banner content max width:** `max-w-5xl mx-auto` to align with page content grid

### Mentions Legales Page

A new public page at `/mentions-legales`. Uses the existing public layout (Header + Footer).

**Page structure:**
- Page title: "Mentions legales" — heading role (28px, weight 700, bleu-nuit)
- Top padding from header: `pt-32` to clear the fixed header
- Content container: `max-w-3xl mx-auto px-4 py-16`
- Sections: h2 headings (heading role), p body text (body role)
- Section spacing: `mt-12` (48px) between sections
- Placeholder values wrapped in brackets: `[A COMPLETER]` — styled `font-mono text-rouge-erreur` to make them visually obvious for Antoine

**Sections declared (in order):**
1. Editeur du site (nom, SIRET, adresse, email)
2. Hebergement (Vercel Inc., San Francisco, USA)
3. Directeur de publication (Antoine Profit)
4. Donnees personnelles et RGPD
5. Cookies et traceurs
6. Photos et consentement clients
7. Contact

**Admin editability:** Page content served from Payload Global "MentionsLegales" (rich text). The page component fetches this Global server-side, same pattern as ContactSettings.
