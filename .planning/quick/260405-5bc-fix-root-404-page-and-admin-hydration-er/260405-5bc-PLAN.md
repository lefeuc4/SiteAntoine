**Step 1: Rewrite `src/app/layout.tsx` to be a minimal pass-through.**

Replace the entire file with a minimal root layout that ONLY renders `{children}` — no `<html>`, no `<body>`, no font imports, no globals.css, no Analytics, no CookieBanner. Keep only the metadata export (metadataBase, title, description) since Next.js metadata works at any layout level.

IMPORTANT: Next.js requires the root layout to return `<html>` and `<body>` tags — this is a hard requirement. So instead of a bare pass-through, the root layout should render minimal `<html><body>{children}</body></html>` WITHOUT any of the frontend-specific additions (no fonts, no globals.css, no Analytics, no CookieBanner). This way Payload's RootLayout... wait, that would still cause nesting.

CORRECT APPROACH: The standard Payload 3 pattern is to have the root `src/app/layout.tsx` export only `{children}` as a React fragment — literally just `return <>{children}</>`. This works because each route group layout (`(frontend)/layout.tsx` and `(payload)/layout.tsx`) provides its own `<html>` and `<body>`. Next.js allows this pattern when every route group has its own root-level html/body layout.

Root layout (`src/app/layout.tsx`):
```tsx
import React from 'react'

// Metadata stays here — applies to all routes
export const metadata = {
  metadataBase: new URL(process.env.NEXT_PUBLIC_SERVER_URL || 'https://antoineprofit.com'),
  title: {
    default: 'Antoine Profit — Coach Bien-Etre',
    template: '%s — Antoine Profit',
  },
  description: 'Coaching bien-etre, nutrition et transformation physique avec Antoine Profit',
}

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return children
}
```

Note: `return children` (not wrapped in Fragment) — this is the Payload 3 documented pattern. The `children` type annotation handles this. If TypeScript complains, cast: `return children as React.JSX.Element`.


This ensures:
